# Uptime Kuma — Мониторинг доступности сервисов

## Что такое Uptime Kuma?

**Uptime Kuma** — это современный инструмент для мониторинга доступности сервисов и веб-сайтов с красивым веб-интерфейсом. Она позволяет отслеживать состояние ваших локальных и удалённых сервисов, получать уведомления о сбоях и просматривать статистику доступности.

## Зачем нужна Kuma в локальном использовании?

### Основные преимущества:

1. **Мониторинг домашней лаборатории** — отслеживайте доступность всех ваших локальных сервисов (DNS-сервер, AdGuard Home, NAS, веб-панели и т.д.)

2. **Контроль интернет-соединения** — проверяйте стабильность подключения к разным провайдерам и внешним ресурсам (Google, Yandex, и др.)

3. **Быстрое обнаружение проблем** — мгновенно узнавайте о сбоях в вашей сети до того, как они станут критическими

4. **Статистика и история** — получайте детальную статистику по времени доступности (uptime) каждого сервиса

5. **Гибкие типы проверок**:
   - HTTP/HTTPS запросы
   - Ping (ICMP)
   - Проверка портов (TCP)
   - DNS запросы
   - И многие другие

### Примеры использования в локальной сети:

- Мониторинг DNS-сервера (как в конфигурации: `192.168.3.1`)
- Проверка доступности панели управления роутером/Nginx Proxy Manager
- Отслеживание работы AdGuard Home (порт 5055)
- Контроль доступности внешних ресурсов для диагностики интернета

## Требования

- Docker и Docker Compose
- Порт `3001` (или любой другой на ваш выбор)

## Установка и запуск

### 1. Клонирование/подготовка

Убедитесь, что вы находитесь в папке с файлом `docker-compose.yml`:

```bash
cd /path/to/kuma
```

### 2. Запуск контейнера

```bash
docker compose up -d
```

Команда запустит контейнер в фоновом режиме.

### 3. Первоначальная настройка

1. Откройте браузер и перейдите по адресу: **http://localhost:3001**
2. Создайте учётную запись администратора
3. Настройте мониторы согласно вашим потребностям

### 4. Остановка

```bash
docker compose down
```

### 5. Просмотр логов

```bash
docker compose logs -f
```

## Конфигурация из данного репозитория

### Преднастроенные мониторы

В файле `Uptime_Kuma_Backup.json` сохранена конфигурация со следующими мониторами:

| № | Название | Тип | Объект мониторинга | Интервал |
|---|----------|-----|-------------------|----------|
| 1 | Basic internet (google.com) | HTTP | https://www.google.com/ | 120 сек |
| 2 | Local DNS server | DNS | 192.168.3.1:53 | 120 сек |
| 3 | Nginx Proxy Manager | Port | 192.168.3.1:81 | 60 сек |
| 4 | Ru internet (yandex.ru) | HTTP | https://yandex.ru | 120 сек |
| 5 | Ping Google.com | Ping | google.com | 120 сек |
| 6 | Ping Yandex.ru | Ping | yandex.ru | 120 сек |
| 7 | Adguard Home | Port | 192.168.3.1:5055 | 120 сек |

### Восстановление из резервной копии

Для применения настроек из `Uptime_Kuma_Backup.json`:

1. Запустите Uptime Kuma
2. Войдите под учётной записью администратора
3. Перейдите в **Settings → Backup/Restore**
4. Нажмите **Restore** и выберите файл `Uptime_Kuma_Backup.json`

## Настройка Docker Compose

### Изменение порта

Если порт `3001` занят, измените в `docker-compose.yml`:

```yaml
ports:
  - "8080:3001"  # Доступ по http://localhost:8080
```

### Постоянный перезапуск

Контейнер настроен на автоматический перезапуск при сбоях:

```yaml
restart: always
```

### Сохранение данных

Данные сохраняются в папку `./uptime-kuma-data` относительно текущего каталога.

## Дополнительные возможности

### Уведомления

Uptime Kuma поддерживает уведомления через:
- Telegram
- Discord
- Slack
- Email (SMTP)
- Pushover
- И многие другие сервисы

Настройка осуществляется через веб-интерфейс: **Settings → Notifications**

### Статусная страница

Создайте публичную статусную страницу для отображения состояния сервисов:
- **Settings → Status Pages → Add New Status Page**

### Автоматический бэкап

Рекомендуется периодически экспортировать настройки:
- **Settings → Backup/Restore → Backup**

## Безопасность

- Используйте надёжные пароли для учётной записи
- При доступе из внешней сети настройте HTTPS (через Nginx Proxy Manager или аналогичный)
- Ограничьте доступ к порту `3001` на уровне файрвола
