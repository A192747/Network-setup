# Qwen Code CLI в Docker контейнере

Данная инструкция описывает процесс развёртывания и запуска **Qwen Code CLI** в Docker контейнере с возможностью подключения по SSH.

## 📋 Требования

- Docker и Docker Compose (опционально)
- SSH клиент для подключения
- Доступ к интернету для загрузки образа и установки пакетов

## 🏗️ Структура проекта

```
qwen-cli/
├── Dockerfile          # Инструкция по сборке Docker образа
├── start.sh           # Скрипт запуска контейнера
└── README.md          # Этот файл
```

## 🚀 Быстрый старт

### Шаг 1: Сборка Docker образа

```bash
docker build -t qwen-cli-ssh .
```

### Шаг 2: Запуск контейнера

```bash
docker run -d \
  --name qwen-cli \
  -p 2222:22 \
  -v $(pwd)/projects:/home/qwenuser/projects \
  -v qwen-data:/home/qwenuser/.qwen \
  --restart unless-stopped \
  qwen-cli-ssh:latest
```

**Параметры запуска:**
- `-p 2222:22` — пробрасывает порт 2222 хоста на порт 22 контейнера (SSH)
- `-v $(pwd)/projects:/home/qwenuser/projects` — монтирует локальную папку `projects` в контейнер
- `-v qwen-data:/home/qwenuser/.qwen` — создаёт именованный том для хранения данных Qwen
- `--restart unless-stopped` — автоматический перезапуск контейнера при сбое

## 🔧 Настройка после запуска

После запуска контейнера выполните следующие команды **внутри контейнера**:

### 1. Подключение к контейнеру

```bash
docker exec -it -u root qwen-cli bash
```

### 2. Настройка прав доступа к директории Qwen

```bash
sudo chown -R qwenuser:qwenuser /home/qwenuser/.qwen
sudo chmod -R 755 /home/qwenuser/.qwen
```

### 3. Запуск Qwen Code CLI

```bash
qwen
```

### 4. Авторизация

Выберите вариант подключения **Qwen Auth** и следуйте дальнейшей инструкции по аутентификации.

## 📡 Подключение по SSH

Для подключения к контейнеру извне (с хост-машины или другой машины):

```bash
ssh -p 2222 qwenuser@localhost
```

**Учётные данные:**
- **Логин:** `qwenuser`
- **Пароль:** `rootpasswd`

> ⚠️ **Важно:** После первого подключения рекомендуется сменить пароль по умолчанию.

## 📁 Тома данных

| Том | Путь в контейнере | Назначение |
|-----|-------------------|------------|
| `./projects` | `/home/qwenuser/projects` | Ваши проекты (синхронизация с хостом) |
| `qwen-data` | `/home/qwenuser/.qwen` | Данные Qwen (настройки, кэш, сессии) |

## 🛠️ Управление контейнером

### Просмотр статуса

```bash
docker ps -f name=qwen-cli
```

### Просмотр логов

```bash
docker logs qwen-cli
```

### Остановка контейнера

```bash
docker stop qwen-cli
```

### Запуск остановленного контейнера

```bash
docker start qwen-cli
```

### Удаление контейнера

```bash
docker rm -f qwen-cli
```

### Удаление образа

```bash
docker rmi qwen-cli-ssh
```

### Удаление именованного тома

```bash
docker volume rm qwen-data
```

## 🔒 Рекомендации по безопасности

1. **Смените пароль по умолчанию** внутри контейнера:
   ```bash
   passwd
   ```

2. **Не используйте порт 2222** в продакшене без необходимости — выберите нестандартный порт.

3. **Настройте аутентификацию по SSH-ключам** вместо паролей для повышения безопасности.

4. **Ограничьте доступ к порту SSH** с помощью фаервола, если контейнер доступен из внешней сети.

## 🐛 Решение проблем

### Контейнер не запускается

Проверьте логи:
```bash
docker logs qwen-cli
```

### Ошибка подключения по SSH

Убедитесь, что порт 2222 не занят:
```bash
netstat -tlnp | grep 2222
```

### Сброс данных Qwen

Удалите именованный том и перезапустите контейнер:
```bash
docker volume rm qwen-data
docker restart qwen-cli
```
